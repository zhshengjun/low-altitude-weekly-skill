# 总结提示词

用于生成《本周热点分析》的评论式分析、目录、栏目分析和结论。输出中文，避免空话。面向政务研究材料时，采用稳健、正式、可进入政策简报的表述。

## 单条资讯评论式分析

对每条资讯生成 `ai_summary` 时使用：

```text
请基于以下资讯，写一篇中文评论式分析。
写法参考研究简报，不要写成新闻摘要或标题改写。
输入中的原文标题、来源、链接、发布时间只用于内部核验和事实理解，不得出现在最终正文中。

必须覆盖五点：
1. 事实触发：本周期出现了什么关键变化。
2. 核心判断：它反映了“{领域}”的什么趋势或问题。
3. 重要性：为什么这件事值得纳入本周分析。
4. 约束或风险：当前仍有哪些不确定、短板或执行难点。
5. 关注变量：后续最应观察什么指标、政策动作、项目进度或主体行为。

正文要求：
- 如果进入“分析专栏”，写 3 到 5 个自然段，正文不少于 500 字。
- 如果进入“优选信息”，写 2 到 4 个自然段，正文不少于 250 字。
- 开头直接进入事实和判断，不写“某媒体某日发布/刊发《某文》”。
- 不输出来源名称、发布时间、原文标题、URL、Markdown 链接或“来源：...；发布时间：...”。
- 不写“这条信息”“这类信息”“该信息”“这条资讯”“该资讯”“信息的价值在于”“信息的重要性在于”等资讯播报腔。
- 不写“约束也很清楚”“约束在于”“风险在于”“限制在于”“真正的执行难点在于”“这一变化值得重视”等模板衔接句；把约束、风险和重要性自然并入正文。
- 不要写“观点：”，不要写“最终分析结论”，不要写“总体判断”，不要写“AI分析：”。

不要加入原文没有支持的判断；可以基于已给事实做审慎推论，但必须写清依据。用户明确指定地域、机构、政策视角或应用场景时，围绕该口径说明影响；用户未指定时，不自行套入固定地区或行业偏好。

原文标题（仅供内部理解，不得照抄到正文）：{title}
已核验来源（仅供内部核验，不得展示）：{source_name}
已核验链接（仅供内部核验，不得展示）：{source_url}
已核验发布时间（仅供内部核验，不得展示）：{published_at}
正文要点：{content_points}
```

## 多篇资讯合并分析

多篇文章讨论同一事件、同一政策或同一主题簇时使用：

```text
请基于以下多篇资讯，写一篇中文评论式分析。
不要逐条拼接摘要；先提炼共同信号，再给出判断。
资讯列表中的原文标题、来源、链接、发布时间只用于内部核验和事实理解，不得出现在最终正文中。

必须说明：
1. 多篇资讯共同指向的事实变化。
2. 这个变化反映的结构性趋势、治理问题、产业变化或区域竞争。
3. 不同来源之间是否存在口径差异；如无差异，不必硬写。
4. 对“{领域}”的实际影响和约束。
5. 下一步最值得关注的变量。

正文写 3 到 5 个自然段，不少于 600 字。不要写“某媒体某日发布/刊发《某文》”，不要输出来源名称、发布时间、原文标题、URL 或 Markdown 链接。不要写“观点：”，不要写“最终分析结论”，不要写“总体判断”。不要写“约束也很清楚”“约束在于”“风险在于”等模板衔接句。

领域：{领域}
主题：{topic}
资讯列表：{items}
```

## 栏目分析

每个栏目或主题开头写 1 段综合分析时使用：

```text
请基于以下同一栏目或主题资讯，写一段 250 到 400 字中文综合分析。
说明本类资讯共同反映的趋势、关键变化、约束条件和可能影响。
不要逐条复述标题；要提炼共同主题和本周增量。
不要输出来源名称、发布时间、原文标题、URL 或 Markdown 链接。
不要使用“约束也很清楚”“约束在于”“风险在于”等模板衔接句。

领域：{领域}
栏目或主题：{category}
资讯列表：{items}
```

## 目录生成

报告开头的目录使用：

```text
请基于本周全部资讯，生成一份简短目录。
目录分为“分析专栏”和“优选信息”两组。
“分析专栏”列 1 到 3 个最重要主题，标题必须体现判断或问题意识。
“优选信息”列代表性资讯，按“政策、形势、行业、企业、技术、项目、风险”等贴切标签标注。
所有标题都必须是编辑生成的主题标题，不照抄原文标题，不出现来源、发布时间或链接。先确定正文各条 `〖〗` 标题，再把同一组标题复制到目录；正文没有对应条目的标题不得进入目录。
不要写摘要句，只列条目；不得留空编号。

领域：{领域}
统计周期：{开始日期} 至 {结束日期}
分类与资讯：{classified_items}
```

## 最终报告组装

生成最终正文时使用；这是默认交付格式，不得省略标题、目录或正文条目标题：

```text
请基于已写好的分析专栏和优选信息，组装最终《本周热点分析》正文。

先为每个正文条目确定一个 `display_title`：
- 标题必须是编辑生成的主题标题，不照抄原文标题。
- 分析专栏标题不加标签，例如 `公共服务投入成为扩大内需的关键前提`。
- 优选信息标题可加标签，例如 `政策：社会救助法强化低收入群体兜底保障`。
- 标题控制在 4 到 40 个汉字左右，不要把正文首句当标题。
- 如果某条正文还没有标题，必须从正文第一段提炼一个标题，不得留空。

最终正文必须严格使用下面骨架：

《{领域}本周热点分析》

统计周期：{开始日期}至{结束日期}

目录

〖分析专栏〗
1. {分析专栏display_title_1}
2. {分析专栏display_title_2}

〖优选信息〗
1. {优选信息display_title_1}
2. {优选信息display_title_2}

〖分析专栏〗

〖{分析专栏display_title_1}〗

{分析正文_1}

〖{分析专栏display_title_2}〗

{分析正文_2}

〖优选信息〗

〖{优选信息display_title_1}〗

{优选正文_1}

〖{优选信息display_title_2}〗

{优选正文_2}

硬性要求：
- 目录编号后必须有标题；不得输出空编号。
- 正文每条必须有单独一行完整 `〖〗` 标题。
- `〖〗` 标题必须独占一行，必须以 `〖` 开头、以 `〗` 结尾；标题后空一行再写正文。
- 不得把标题和正文粘在同一行，例如不得写成 `〖标题正文第一句...`。
- 标题不得过长，不能像正文首句；控制在 4 到 40 个汉字左右。
- 目录标题必须和正文 `〖〗` 标题逐项一致；目录可不带 `〖〗`，但文字必须一致。
- 目录里的每一个标题都必须在正文中重复一次并包上 `〖〗`；不能只把标题放在目录里。
- 正文条目不得直接从段落开始；段落前必须先写对应的 `〖display_title〗`。
- 不输出任何链接、URL、Markdown 链接、来源名称、发布时间、原文标题或来源区块。
- 不输出 `来源链接`、`核验来源`、`参考来源`、`来源列表`、`来源明细表`。
- 如果草稿出现上述任何硬性问题，先修正后再输出，不要解释。

领域：{领域}
统计周期：{开始日期}至{结束日期}
分析专栏：{analysis_items}
优选信息：{selected_items}
```

## 本周态势判断

生成分析专栏和优选信息前，先基于已入选资讯做轻量态势识别。只能引用已入选资讯，不允许根据常识或外部猜测补判断。

```text
请基于以下已入选资讯，判断本周“{领域}”是否出现明显态势。

可选态势：
- 政策窗口期：政策、监管、标准、规划、通知等资讯占比较高，或同一方向政策连续发布。
- 项目推进密集期：项目、投资、招标、中标、开工、投产、签约等资讯集中出现。
- 产业活跃期：企业、产品、融资、合作、订单、技术发布等资讯集中出现。
- 风险暴露期：处罚、事故、整改、争议、预警、投诉、召回等资讯集中出现。
- 常态跟踪期：以上特征都不明显。

输出 2 到 4 句话：
1. 给出态势标签。
2. 说明判断依据，必须点出对应资讯类型或代表性事件。
3. 说明下一步最值得关注的变量。

领域：{领域}
统计周期：{开始日期} 至 {结束日期}
已入选资讯：{items}
```

## 报告自检与修正

输出最终报告前使用：

```text
请检查以下《本周热点分析》草稿，只修正问题部分，不要重写全文。

检查项：
1. 是否包含标题、统计周期、目录、分析专栏、优选信息，且不包含总体判断。
2. 每条资讯是否已在内部核验来源名称、来源链接、发布时间和领域相关性。
3. 正文是否隐藏来源名称、发布时间、原文标题、URL 和 Markdown 链接。
4. 正文是否采用附件式连续段落，而不是 bullet 摘要或“AI分析：”字段。
5. 正文中每条是否都有单独一行 `〖〗` 标题，且左右括号成对闭合。
6. 每条开头是否直接进入正文，且核心判断由正文事实支撑。
7. 每条分析是否为充分展开的评论式分析，而不是简单摘要、标题改写、“观点：”字段、“这条信息...”式播报腔或“约束在于...”式模板腔。
8. 分析专栏是否为 1 到 3 条；是否存在同一资讯重复进入多个栏目。
9. 是否存在“最终分析结论”或“总体判断”；如有必须删除。
10. 目录是否存在空白编号，例如 `1.`、`1.  `、`2.` 后没有标题；如有必须补成正文标题或删除。
11. 目录标题是否与正文 `〖〗` 编辑主题标题逐项一致。
12. 是否存在 `http`、`www.`、`](`、`来源链接`、`核验来源`、`参考来源`、`来源列表`、`来源明细表`；如有必须删除。
13. 是否存在搜索结果页、转载页、无发布时间资讯或无来源预测。

如果发现问题：
- 缺少固定层级：补齐该层级。
- 格式过于周报化或列表化：改为“目录 + 〖分析专栏〗 + 〖优选信息〗”。
- 分析过于摘要化：改成连续段落，并补充事实触发、核心判断、影响链条、约束条件和关注变量；分析专栏单篇不少于 500 字，优选信息单篇不少于 250 字。
- 分类过散或过粗：重新合并或拆分分类。
- 判断无证据：删除或降级，不要补造依据。
- 来源或发布时间缺项：补齐内部核验记录；无法补齐则从正文移除。默认正文不要补成逐条“来源：...；发布时间：...”行。
- 正文出现来源名称、发布时间、原文标题、URL、Markdown 链接或“来源链接”：改写为不带元数据的事实表述。
- 正文出现“这条信息/这类信息/该资讯的价值在于”等播报腔：删掉指代，改为直接陈述事实变化、判断和影响。
- 正文出现“约束也很清楚/约束在于/风险在于/限制在于”等模板衔接句：删除模板句，把风险和限制合并进具体分析。
- 目录有空白编号：不得保留空项；先为正文条目补完整 `〖〗` 标题，再把标题复制到目录。
- 正文条目缺少 `〖〗` 标题、标题括号残缺、标题和正文粘连，或标题过长像正文首句：补成 4 到 40 个汉字左右的完整编辑主题标题；标题必须单独成行，下一空行再写正文。
- 目录标题和正文 `〖〗` 标题不一致：以正文标题为准重建目录。
- 出现末尾来源区块或任何链接：删除；来源只保留在内部核验记录。

草稿：{draft}
已入选资讯：{items}
```

## 禁止事项

- 不要编造数据、发布日期、来源或链接。
- 不要把“值得关注”“影响深远”等空话当结论。
- 不要把政务研究材料写成投资建议、营销文案或媒体评论。
- 不要输出没有证据支撑的预测。
- 不要把同一条资讯写进多个分类。
- 不要只因出现一条政策、项目或风险资讯就判断进入窗口期；必须有集中度或代表性事件支撑。
- 不要把单条资讯写成简单摘要；必须有判断、影响、约束和关注变量。
- 不要写“观点：”“最终分析结论”或“总体判断”；每条正文在首段自然承接判断。
- 不要在最终正文中输出来源名称、发布时间、原文标题、URL 或 Markdown 链接。
- 不要写“这条信息”“这类信息”“该信息”“这条资讯”“该资讯”等资讯播报腔。
- 不要写“来源链接”“核验来源”“参考来源”“来源列表”等末尾来源区块。
- 不要写“约束也很清楚”“约束在于”“风险在于”“限制在于”等模板衔接句。
